# WorkBuddy 会话管理设计

## 目标

在 **WebDAV 同步** 左侧新增 **会话管理** Tab，供用户查看、搜索和软删除本机 WorkBuddy 会话。删除操作保留回收数据，避免直接永久删除会话文件。

## 页面与交互

页签顺序为：**模型配置**、**供应商管理**、**会话管理**、**WebDAV 同步**。

会话列表仅显示 `workbuddy.db` 的 `sessions` 表中未删除的记录。每条记录显示：

- 标题
- 工作目录
- 模型
- 状态
- 最后活动时间
- 会话文件大小

页面支持按标题或工作目录搜索，并提供刷新按钮。状态为 `working` 的会话禁用删除按钮。其他会话删除前必须二次确认。

浏览器预览环境返回空列表；删除操作返回仅桌面端可用的提示。

## 删除流程

删除操作按以下顺序执行：

1. 校验会话存在、尚未删除且状态不是 `working`。
2. 定位 `projects/<cwd-key>/<session-id>.jsonl` 及该会话的同名子代理目录。
3. 将存在的文件和目录移动到 `.workbuddy/session-trash/<timestamp>/<session-id>/`，并保留可回移所需的原路径信息。
4. 在 `workbuddy.db` 中设置该会话的 `deleted_at`。
5. 从 `app/sessions.json` 的最近会话索引中移除该会话。
6. 返回成功结果并刷新列表。

如果数据库更新失败，后端将已移动的文件回移到原位置，再返回错误。文件移动失败时不得更新数据库。

## 数据与安全边界

- 不删除 `.workbuddy/sessions/<pid>.json` 等运行时进程文件。
- 不直接操作 `workbuddy.db-wal` 或 `workbuddy.db-shm`。
- 不解析或改写 JSONL 消息正文。
- 不允许删除 `working` 状态的会话。
- 本期不提供恢复会话或清空回收站功能。
- `app/sessions.json` 更新失败不应破坏数据库与回收站的一致性；实现应采用安全写入，并向调用方返回明确错误。

## 接口边界

后端提供两个桌面命令：

- `list_workbuddy_sessions`：读取未删除会话，并补充对应 JSONL 文件大小。
- `delete_workbuddy_session`：接收会话 ID，执行校验、移入回收站、数据库软删除和最近会话索引清理。

前端只通过命令访问会话数据，不直接读取数据库或文件系统。
