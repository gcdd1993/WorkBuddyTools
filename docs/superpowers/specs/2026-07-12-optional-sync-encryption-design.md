# WebDAV 可选同步加密设计

## 目标

让同步加密密码成为可选项，并在界面中说明配置档案和同步加密密码的作用。用户填写密码时上传加密包；留空时上传明文 ZIP，并获得明确的安全警告。

## 用户界面

- 在**配置档案**标签旁增加感叹号图标。悬浮后说明：配置档案用于区分不同的同步空间，默认值为 `default`。
- 在**同步加密密码**标签旁增加感叹号图标。悬浮后说明：填写密码会上传加密包；留空会上传明文包。
- 同步加密密码不再必填。
- 当密码为空时，页面显示醒目的风险提示：会话、模型配置和其中可能包含的 API Key 将以明文保存在 WebDAV。
- 当密码不为空时，沿用当前的加密上传说明。

## 上传行为

同步包的内容和生成流程保持不变，仅根据同步加密密码选择上传格式：

| 密码状态 | 远端文件名 | 处理方式 |
| --- | --- | --- |
| 已填写 | `workbuddy-sync.zip.enc` | 加密 ZIP 后上传 |
| 留空 | `workbuddy-sync.zip` | 直接上传明文 ZIP |

`manifest.public.json` 和 `latest.json` 中的包路径、大小与校验值必须对应实际上传文件。代码不得再假定远端文件名固定为 `workbuddy-sync.zip.enc`。

## 下载行为

下载端读取清单中的实际包路径：

- 路径以 `.zip.enc` 结尾时，要求同步加密密码，并在解密后导入。
- 路径以 `.zip` 结尾时，直接校验并导入，无需密码。
- 其他后缀视为不支持的同步包，返回清晰错误。

已有的 `workbuddy-sync.zip.enc` 远端包继续按原逻辑下载和解密，确保向后兼容。

## 涉及文件

- `src/main.tsx`：标签提示、密码可选状态与动态风险警告。
- `src/styles.css`：感叹号图标、悬浮提示和警告样式。
- `src-tauri/src/webdav.rs`：根据密码选择明文或加密上传，并按远端路径选择下载处理方式。

## 安全边界

明文模式是用户主动留空密码后的显式行为。界面必须紧邻输入区域说明风险，但不阻止上传。WebDAV 登录密码仅用于服务认证，不作为同步包的隐式加密密码。
