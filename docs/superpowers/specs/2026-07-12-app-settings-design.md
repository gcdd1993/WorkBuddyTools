# WorkBuddy Tools 应用设置设计

## 目标

为 WorkBuddy Tools 增加独立设置页，统一管理本工具使用的完整 **WorkBuddy 数据目录** 与 WebDAV 连接参数。设置保存后立即影响模型、供应商、会话管理及备份同步功能。

## 页面入口与导航

在页面顶部操作区增加齿轮按钮。点击后打开独立设置页，设置页提供返回主界面的入口。

原 **WebDAV 同步** Tab 更名为 **备份与同步**。WebDAV 地址、用户名、密码、远端目录和同步加密密码从该 Tab 移入设置页；Tab 仅保留同步策略、远端版本信息、风险说明和测试连接、查看远端、执行同步、覆盖上传或下载等操作。

## 设置项

设置页包含：

- WorkBuddy 数据目录
- WebDAV 地址
- WebDAV 用户名
- WebDAV 密码或 Token
- WebDAV 远端目录
- 同步加密密码（可留空；留空时按现有规则上传明文 ZIP）

WorkBuddy 数据目录应指向包含 `models.json`、`model-providers.json`、`workbuddy.db`、`projects` 等数据的完整目录。保存后，模型、供应商、会话和同步均使用新目录；不复制或迁移任何数据，也不会修改 WorkBuddy 主程序自身的数据目录。页面必须明确提示这两个限制。

WebDAV 密码和同步加密密码按用户要求直接明文保存在本机设置文件中。设置页应显示风险提示，说明能够读取该文件的程序或用户可以获得凭据。

## 本机设置存储

设置保存到本工具本机配置目录中的 `settings.json`。建议结构如下：

```json
{
  "workbuddyDir": "C:\\Users\\user\\.workbuddy",
  "webdav": {
    "baseUrl": "https://dav.example.com/",
    "username": "user",
    "password": "plain-text-password",
    "remoteRoot": "WorkBuddySync",
    "passphrase": "plain-text-passphrase"
  }
}
```

后端负责加载和保存设置。保存采用同目录临时文件写入并原子替换，避免进程中断产生半写入文件。首次启动或设置文件不存在时返回默认设置；无效 JSON 或不可用路径应返回明确错误，不静默覆盖原文件。

## 数据目录解析优先级

所有后端功能统一通过 `workbuddy_dir` 获取数据目录，优先级为：

1. `settings.json` 中非空的 `workbuddyDir` 覆盖值
2. `WORKBUDDY_HOME` 环境变量
3. `%USERPROFILE%\.workbuddy`

保存设置后，后续命令立即按此优先级读取，无需重启应用。

## 前端状态与刷新

应用启动时加载设置，并以其初始化设置页和备份同步状态。保存成功后调用统一刷新流程，重新加载模型、供应商、会话、路径展示及远端信息，确保新目录立即生效。

设置页输入在保存前仅为草稿，不应提前改变其他页面的数据来源。保存失败时保留用户输入并显示错误。

## 浏览器预览

浏览器预览环境为加载设置提供与字段结构一致的默认值；保存操作仅更新预览期内存状态或返回明确的桌面端限制，不访问真实文件系统。其他既有预览命令继续使用合理的空数据或桌面端提示。

## 安全与边界

- 凭据明文落盘是明确的产品选择，必须在设置页提示风险。
- 本工具不自动修改 WorkBuddy 的启动参数、环境变量或数据位置。
- 切换目录不迁移、合并或删除原目录数据。
- 不将应用设置文件打入现有会话同步包；同步仍仅包含既定的 WorkBuddy 会话与模型配置。
- 路径输入需要去除首尾空白，并在保存或首次使用时给出可理解的路径错误。
