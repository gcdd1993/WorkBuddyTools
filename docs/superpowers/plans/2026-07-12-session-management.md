# WorkBuddy 会话管理实施计划

## 1. 增加后端会话模型与查询命令

在 Tauri 后端新增会话管理模块：

- 定义会话列表项和删除结果的数据结构。
- 以只读方式查询 `workbuddy.db` 的 `sessions` 表，仅返回 `deleted_at` 为空的记录。
- 为每条记录定位项目 JSONL，并计算文件大小。
- 注册 `list_workbuddy_sessions` 命令。
- 保持数据库、WAL、SHM 和 PID 运行时文件不受影响。

## 2. 实现软删除与回滚

在同一模块实现 `delete_workbuddy_session`：

- 校验会话状态，拒绝删除 `working` 会话。
- 将主 JSONL 和同名子代理目录移动到 `.workbuddy/session-trash/<timestamp>/<session-id>/`。
- 文件移动成功后，在数据库事务中设置 `deleted_at`。
- 数据库更新失败时，将文件和目录回移到原位置。
- 从 `app/sessions.json` 的最近会话索引中移除会话，并使用临时文件替换方式安全写入。
- 返回清晰的成功或失败信息。

## 3. 增加前端运行时命令适配

修改前端 Tauri 调用适配层：

- 桌面端调用新增的列表和删除命令。
- 浏览器预览的列表命令返回空数组。
- 浏览器预览的删除命令返回“仅桌面端可用”错误。

## 4. 增加会话管理页面

修改主界面：

- 在 **WebDAV 同步** 左侧加入 **会话管理** Tab。
- 加载并展示标题、工作目录、模型、状态、最后活动时间和文件大小。
- 增加标题或工作目录搜索。
- 增加刷新按钮和加载、空数据、错误状态。
- 对 `working` 会话禁用删除按钮。
- 删除前显示二次确认；成功后刷新列表。

## 5. 补充页面样式

修改现有样式表：

- 将顶部页签布局扩展为四项。
- 增加搜索栏、会话列表、状态标记和操作按钮样式。
- 处理长标题、长工作目录和窄窗口下的换行或截断。
- 保持现有暗色主题与交互状态一致。

## 完成条件

- **会话管理** 位于 **WebDAV 同步** 左侧。
- 列表只显示数据库中未删除的会话，并支持搜索和刷新。
- `working` 会话无法删除。
- 删除后，会话文件进入回收站，数据库记录写入 `deleted_at`，最近会话索引移除对应项。
- 数据库更新失败时，已移动文件能够回到原位置。
- 浏览器预览不会访问本地数据库或删除本地文件。
- 未实现恢复会话和清空回收站。
