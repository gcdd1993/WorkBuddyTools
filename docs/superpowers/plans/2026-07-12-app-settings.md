# WorkBuddy Tools 应用设置实施计划

## 1. 增加后端应用设置模块

- 定义应用设置与 WebDAV 设置的数据结构及默认值。
- 确定本工具配置目录和 `settings.json` 路径。
- 实现加载设置命令：文件不存在时返回默认值，解析失败时返回明确错误。
- 实现保存设置命令：规范化输入，使用临时文件和原子替换写入。
- 注册 Tauri 命令，并保持密码字段按要求明文读写。

## 2. 统一 WorkBuddy 数据目录解析

- 将 `workbuddy_dir` 改为读取应用设置覆盖值。
- 严格实现 `settings override > WORKBUDDY_HOME > USERPROFILE/.workbuddy` 的优先级。
- 确保模型、供应商、会话管理、同步打包、恢复和备份等现有功能继续通过统一入口获取目录。
- 切换目录时不执行复制、迁移、合并或删除操作。

## 3. 增加前端设置状态与运行时适配

- 为应用设置定义前端类型、默认值和加载状态。
- 应用启动时调用后端加载命令。
- 保存设置后更新前端状态并调用 `refreshAll`，重新加载模型、供应商、会话、路径展示和远端信息。
- 为浏览器预览增加结构一致的默认设置及合理的保存 fallback，不访问本机文件系统。

## 4. 实现独立设置页

- 在顶部操作区增加齿轮按钮，点击打开设置页。
- 增加返回主界面的操作。
- 提供完整 WorkBuddy 数据目录、WebDAV 地址、用户名、密码或 Token、远端目录和同步加密密码字段。
- 明确显示以下提示：设置只影响本工具；不会改变 WorkBuddy 主程序目录；不会迁移数据；两个密码将明文保存于本机 `settings.json`。
- 增加保存、保存中、成功和失败状态，保存失败时保留草稿。

## 5. 重构备份同步页面

- 将 **WebDAV 同步** Tab 和页面标题改为 **备份与同步**。
- 从 Tab 移除 WebDAV 连接与密码输入框，改为使用已保存设置。
- 保留同步策略、远端版本信息、加密或明文风险说明及所有执行按钮。
- 当必要配置缺失时，显示前往设置页的提示，并禁止或友好拦截依赖配置的操作。

## 6. 调整布局和样式

- 为齿轮按钮、设置页表单、路径提示和明文凭据风险提示增加与现有暗色主题一致的样式。
- 处理长目录、长 WebDAV 地址、窄窗口和键盘焦点状态。
- 保持主 Tab、会话列表和备份同步页面现有滚动行为不受影响。

## 完成条件

- 顶部齿轮按钮可进入独立设置页并返回。
- 设置页可加载和保存完整 WorkBuddy 数据目录及全部 WebDAV 参数。
- 设置以明文 JSON 原子写入本机 `settings.json`，页面明确展示风险。
- 保存目录后，模型、供应商、会话及同步立即使用新目录，且不迁移数据、不改变 WorkBuddy 主程序设置。
- `workbuddy_dir` 遵循设置覆盖值、环境变量、用户目录的既定优先级。
- **WebDAV 同步** 已更名为 **备份与同步**，连接字段只存在于设置页。
- 应用启动加载设置，保存后执行统一刷新。
- 浏览器预览不会访问真实配置文件或本机 WorkBuddy 数据。
